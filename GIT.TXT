GIT MAIN AREAS
Working Directory - Your actual files (what you edit)
Staging Area - Where you prepare changes before committing
Repository - Where commits are permanently stored

GIT FLOW
edit → add → commit → push


GIT COMMANDS
status - Shows: modified files, staged files, untracked files
add - adds a specific file
add . - adds all files/folders
commit -m - a snapshot of your project with a unique ID 

BRANCHING
checkout -b branchname - creates a new branch
switch -c branchname - creates a new branch also
switch - switch between branches 
branch -d - delete branch

clone - 
pull -  fetch latest changes

workflow 
create branch -> pull -> edit -> push branch


PUSHING
push -u origin main - first time push to repo
push - subsequent push to repo

git -u origin main
